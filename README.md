# 芯尘策略计算器

一个可直接部署到 GitHub Pages 的静态网页工具，用于模拟 NIKKE 红球成长过程，并对比不同开箱策略在一段时间内的表现。

## 在线访问

GitHub Pages 地址：

https://tianqing02.github.io/NikkeCoreDustCalculator/

## 项目说明

本项目为纯前端静态页面，不依赖 Python、后端服务或数据库。

当前实现包含：

- 基础参数配置
- 主线更新配置
- 困难推图预测配置
- 活动配置
- 额外来源 / 日常获取配置
- 多策略模拟
- 策略对比图表
- 每日明细查看
- CSV 导出
- PNG 导出

当前内置策略为“获得即开”“完全不开”“主线后开”“最后一天全开”四类。“获得即开”会在开箱升到收益节点后，用更新后的小时芯尘继续计算剩余箱子；“主线后开”会在模拟最后一天补开剩余箱子，避免余量影响对比。

## 目录结构

- `index.html`：页面结构
- `styles.css`：页面样式
- `nikke-data.js`：NIKKE 主线进度与基地等级本地快照
- `app.js`：模拟逻辑与交互逻辑

## 本地使用

直接打开 `index.html` 即可使用。

如果需要更接近线上环境，也可以使用任意静态服务器进行预览。

所有可编辑配置会自动写入浏览器 `localStorage`，刷新页面后会恢复上次配置；模拟结果与图表不会持久化。

额外来源可选择“芯尘”或“芯尘箱”：芯尘会在触发当天直接加入进度，芯尘箱会先进入箱子库存并继续按策略开箱。

活动预设按每日门票进度发放箱子，不会在活动第一天一次性到账；hard 开启第 1 天会用当天门票加一次性买票从第 1 关开始推，通关后剩余门票扫荡 11 关。每日明细中的“活动箱子”和“额外箱子”会分开显示。

## 部署到 GitHub Pages

1. 将 `index.html`、`styles.css`、`app.js`、`nikke-data.js` 提交到仓库。
2. 在仓库 `Settings -> Pages` 中选择部署分支。
3. 如果使用仓库根目录作为发布目录，则入口文件为 `index.html`。
